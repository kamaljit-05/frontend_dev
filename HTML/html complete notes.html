Section 01 · Basics

1. What is HTML?

Definition: HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It uses tags (enclosed in angle brackets) to tell browsers how to display text, images, links, and other elements.

HTML is not a programming language — it is a markup language. HTML5 is the current version and adds semantic elements, multimedia support, and APIs.

Key Terms

TermMeaning
TagAn HTML keyword in angle brackets, e.g. <p>
ElementOpening tag + content + closing tag: <p>Hello</p>
AttributeExtra info inside the opening tag: <a href="...">
Void / Self-closingTags with no closing tag: <br>, <img>
NestingPlacing one element inside another
Section 02 · Basics

2. HTML Document Structure

Definition: Every HTML file must follow a standard skeleton so the browser knows how to parse and render it correctly.
HTML
<!-- Document type declaration -->
<!DOCTYPE html>
<html lang="en">

  <head>
    <!-- Meta info — not displayed on page -->
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My First Page</title>
    <link rel="stylesheet" href="style.css" />
  </head>

  <body>
    <!-- All visible content goes here -->
    <h1>Hello, World!</h1>
    <p>Welcome to my page.</p>
  </body>

</html>
TagPurpose
<!DOCTYPE html>Declares HTML5 mode — always first line
<html>Root element; wrap everything inside
<head>Metadata: title, charset, links — not rendered
<title>Text shown in browser tab
<body>All visible page content

3. Headings & Paragraphs

Heading: Tags <h1>–<h6> define section titles in descending importance. <h1> is the most important (one per page for SEO).

Paragraph: <p> groups a block of text. Browsers add margin above and below automatically.
HTML
<h1>Main Title</h1>
<h2>Section Heading</h2>
<h3>Sub-section</h3>
<h4>Level 4</h4>
<h5>Level 5</h5>
<h6>Level 6 (smallest)</h6>

<p>This is a paragraph of text.</p>
<br>  <!-- Line break (void element) -->
<hr>  <!-- Horizontal rule / divider -->

Sub-section

Section 05 · Basics

5. Links (Anchor Tags)

Definition: The <a> (anchor) tag creates hyperlinks to other pages, files, locations on the same page, or email addresses. The href attribute specifies the destination.
HTML
<!-- External link -->
<a href="https://www.google.com" target="_blank" rel="noopener">
  Visit Google
</a>

<!-- Internal link -->
<a href="about.html">About Page</a>

<!-- Anchor link (jump to section) -->
<a href="#section2">Go to Section 2</a>
<h2 id="section2">Section 2</h2>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Us</a>

<!-- Phone link -->
<a href="tel:+911234567890">Call Us</a>

<!-- Download link -->
<a href="file.pdf" download>Download PDF</a>
AttributeValueMeaning
hrefURL / path / #idDestination of the link
target_blank / _selfWhere to open: new tab or same tab
relnoopener noreferrerSecurity for external _blank links
downloadboolean / filenameForces file download instead of navigation
Section 06 · Basics

6. Images

Definition: The <img> tag embeds an image. It is a void element (no closing tag). The src attribute is required; alt is required for accessibility.
HTML
<!-- Basic image -->
<img src="photo.jpg" alt="A scenic mountain view" />

<!-- With size -->
<img src="logo.png" alt="Company Logo"
     width="200" height="100" />

<!-- Responsive image -->
<img src="hero.jpg" alt="Hero banner"
     style="max-width:100%; height:auto;" />

<!-- Image as link -->
<a href="https://example.com">
  <img src="banner.png" alt="Click here" />
</a>

<!-- Figure with caption -->
<figure>
  <img src="chart.png" alt="Sales chart 2024" />
  <figcaption>Figure 1: Annual Sales Data</figcaption>
</figure>

<!-- Lazy loading -->
<img src="big-image.jpg" alt="..." loading="lazy" />
Always provide a meaningful alt attribute — it improves accessibility and SEO. Use alt="" for decorative images.
Section 07 · Basics

7. Lists

Unordered List (<ul>): Bulleted list — order doesn't matter.
Ordered List (<ol>): Numbered list — order matters.
Description List (<dl>): Term–description pairs (like a glossary).
HTML
<!-- Unordered -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered -->
<ol type="1" start="1">
  <li>Boil water</li>
  <li>Add pasta</li>
  <li>Drain and serve</li>
</ol>

<!-- Nested list -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>

<!-- Description list -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
Section 08 · Intermediate

8. Tables

Definition: HTML tables display data in rows and columns. Use them for tabular data only — not for page layout (use CSS Flexbox/Grid instead).
HTML
<table border="1">
  <caption>Student Scores</caption>

  <thead>
    <tr>
      <th>Name</th>
      <th>Math</th>
      <th>Science</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Ravi</td>
      <td>90</td>
      <td>85</td>
    </tr>
    <tr>
      <td colspan="2">Merged Cell</td>
      <td>70</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td>Average</td>
      <td>90</td>
      <td>78</td>
    </tr>
  </tfoot>
</table>
TagMeaning
<table>Container for the entire table
<thead>Header row group
<tbody>Body/data row group
<tfoot>Footer row group (totals, etc.)
<tr>Table Row
<th>Table Header cell (bold, centered)
<td>Table Data cell
colspanMerge cells horizontally
rowspanMerge cells vertically
Section 09 · Intermediate

9. Forms & Inputs

Definition: HTML forms collect user input and send it to a server. The <form> element wraps all input controls. action sets where data is sent; method sets how (GET or POST).
HTML
<form action="/submit" method="POST">

  <!-- Text input -->
  <label for="name">Full Name:</label>
  <input type="text" id="name" name="name"
         placeholder="Enter name" required />

  <!-- Email -->
  <input type="email" name="email" placeholder="you@email.com" />

  <!-- Password -->
  <input type="password" name="pass" minlength="8" />

  <!-- Number -->
  <input type="number" min="1" max="100" step="1" />

  <!-- Date / Time -->
  <input type="date" name="dob" />
  <input type="time" name="time" />

  <!-- Radio buttons -->
  <input type="radio" name="gender" value="male"/> Male
  <input type="radio" name="gender" value="female"/> Female

  <!-- Checkbox -->
  <input type="checkbox" name="agree" checked/> I agree

  <!-- Dropdown -->
  <select name="city">
    <option value="">-- Select City --</option>
    <option value="bbsr">Bhubaneswar</option>
    <option value="del">Delhi</option>
  </select>

  <!-- Textarea -->
  <textarea name="message" rows="5" cols="40"
            placeholder="Your message..."></textarea>

  <!-- File upload -->
  <input type="file" accept=".jpg,.pdf" multiple />

  <!-- Range slider -->
  <input type="range" min="0" max="100" value="50" />

  <!-- Color picker -->
  <input type="color" value="#ff0000" />

  <!-- Hidden input -->
  <input type="hidden" name="token" value="abc123" />

  <!-- Submit & Reset -->
  <button type="submit">Submit</button>
  <button type="reset">Clear</button>

</form>
Always use <label for="id"> paired with input id — clicking the label focuses the field and improves accessibility.
Section 10 · Intermediate

10. Semantic HTML5 Elements

Definition: Semantic elements clearly describe their purpose to both the browser and the developer. They improve SEO, accessibility, and code readability. Use them instead of generic <div> tags wherever meaning can be expressed.
HTML
<header>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
</header>

<main>
  <article>
    <h2>Blog Post Title</h2>
    <section>
      <p>Introduction paragraph...</p>
    </section>
    <section>
      <p>Body content...</p>
    </section>
  </article>

  <aside>
    Related Links / Sidebar
  </aside>
</main>

<footer>
  <p>&copy; 2024 My Website</p>
</footer>
ElementDefinition
<header>Top of page or section — logo, nav
<nav>Navigation links block
<main>Primary content — one per page
<article>Self-contained content (blog post, card)
<section>Thematic grouping with a heading
<aside>Sidebar / supplementary content
<footer>Bottom of page or section
<figure>Image/media with optional caption
<figcaption>Caption for a <figure>
<time>Dates/times with machine-readable datetime
<details>Expandable/collapsible disclosure widget
<summary>Visible label for <details>
Section 11 · Intermediate

11. Div & Span

<div>: A generic block-level container with no meaning — used for layout/grouping when no semantic tag fits.

<span>: A generic inline container — used to style or target a small piece of text/content inside a block.
HTML
<!-- div: block level wrapper -->
<div class="card">
  <h3>Product Title</h3>
  <p>Description here...</p>
</div>

<!-- span: inline wrapper -->
<p>Price: 
  <span style="color: green; font-weight: bold;">
    ₹499
  </span>
</p>

12. Media – Audio & Video

<audio>: Embeds sound content. Supports MP3, OGG, WAV.
<video>: Embeds video content. Supports MP4, WebM, OGG.
HTML
<!-- Audio -->
<audio controls autoplay loop>
  <source src="audio.mp3" type="audio/mpeg" />
  <source src="audio.ogg" type="audio/ogg" />
  Your browser does not support audio.
</audio>

<!-- Video -->
<video width="640" height="360"
       controls muted poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4" />
  <source src="video.webm" type="video/webm" />
  Your browser does not support video.
</video>

<!-- YouTube embed via iframe -->
<iframe width="560" height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  allowfullscreen></iframe>
Section 13 · Intermediate

13. iFrame

Definition: <iframe> (Inline Frame) embeds another HTML document or external page inside the current page. Commonly used for maps, videos, and payment widgets.
HTML
<!-- Google Maps embed -->
<iframe
  src="https://maps.google.com/maps?q=Bhubaneswar&output=embed"
  width="600"
  height="400"
  style="border:0"
  allowfullscreen
  loading="lazy"
  sandbox="allow-scripts allow-same-origin"
></iframe>
Use the sandbox attribute to restrict iframe permissions and improve security.

15. Meta Tags & SEO

Definition: <meta> tags provide metadata about the document to browsers, search engines, and social platforms. They live in <head> and are not displayed.
HTML
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8" />

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- SEO basics -->
  <meta name="description" content="Learn HTML from basic to advanced." />
  <meta name="keywords" content="HTML, web development, tutorial" />
  <meta name="author" content="Your Name" />

  <!-- Open Graph (social preview) -->
  <meta property="og:title" content="HTML Notes" />
  <meta property="og:description" content="Complete HTML guide" />
  <meta property="og:image" content="https://example.com/preview.jpg" />

  <!-- Twitter card -->
  <meta name="twitter:card" content="summary_large_image" />

  <!-- Robots -->
  <meta name="robots" content="index, follow" />

  <!-- Favicon -->
  <link rel="icon" href="/favicon.ico" type="image/x-icon" />

  <!-- Canonical URL -->
  <link rel="canonical" href="https://example.com/html-notes" />
</head>

Section 17 · Advanced

17. Advanced: Data Attributes

Definition: Custom data-* attributes let you embed extra information in HTML elements without using non-standard attributes. JavaScript can read them via element.dataset.
HTML + JS
<!-- HTML -->
<button
  id="btn"
  data-product-id="42"
  data-price="499"
  data-category="electronics"
>Add to Cart</button>

<!-- JavaScript reading data attributes -->
<script>
  const btn = document.getElementById('btn');
  console.log(btn.dataset.productId);  // "42"
  console.log(btn.dataset.price);      // "499"
  console.log(btn.dataset.category);   // "electronics"

  // You can also set them:
  btn.dataset.inCart = "true";
</script>
Section 18 · Advanced

18. Advanced: Accessibility (ARIA)

Definition: ARIA (Accessible Rich Internet Applications) attributes add accessibility information for screen readers and assistive technologies. Use semantic HTML first — ARIA fills the gaps.
HTML
<!-- Labeling a button with no visible text -->
<button aria-label="Close dialog">✕</button>

<!-- Describing an input -->
<input type="text" aria-describedby="hint" />
<span id="hint">Enter at least 8 characters</span>

<!-- Role -->
<div role="alert">Form submitted successfully!</div>

<!-- Expandable menu -->
<button aria-expanded="false" aria-controls="menu">Menu</button>
<ul id="menu" aria-hidden="true">
  <li>Home</li>
  <li>About</li>
</ul>

<!-- Live region for dynamic updates -->
<div aria-live="polite">Loading results...</div>
ARIA AttributePurpose
roleDefines element's purpose (button, alert, dialog, navigation)
aria-labelAccessible name for an element
aria-describedbyPoints to element that describes this one
aria-hiddenHides element from screen readers
aria-expandedState of collapsible widget
aria-liveAnnounces dynamic content changes
aria-requiredMarks form field as required

19. Advanced: Canvas

Definition: The <canvas> element provides a pixel-based drawing surface controlled entirely by JavaScript. Used for games, charts, image editing, and animations.
HTML + JS
<canvas id="myCanvas" width="400" height="200"
        style="border: 1px solid #ccc;">
</canvas>

<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');

  // Draw a filled rectangle
  ctx.fillStyle = '#4caf50';
  ctx.fillRect(20, 20, 150, 100);

  // Draw a circle
  ctx.beginPath();
  ctx.arc(300, 100, 60, 0, Math.PI * 2);
  ctx.fillStyle = '#2196f3';
  ctx.fill();

  // Draw text
  ctx.font = '20px Arial';
  ctx.fillStyle = '#fff';
  ctx.fillText('Canvas!', 250, 108);

  // Draw a line
  ctx.beginPath();
  ctx.moveTo(20, 180);
  ctx.lineTo(380, 180);
  ctx.strokeStyle = '#f44336';
  ctx.lineWidth = 3;
  ctx.stroke();
</script>

JS – Geolocation Example
navigator.geolocation.getCurrentPosition(
  (position) => {
    const lat = position.coords.latitude;
