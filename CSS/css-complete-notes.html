Chapter 01

What is CSS?

Definition: CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of an HTML document — controlling colors, fonts, spacing, layout, and animation.

CSS separates content (HTML) from design (CSS), making code cleaner and reusable.

Three Ways to Add CSS

<!-- 1. Inline -->
<p style="color: red;">Hello</p>

<!-- 2. Internal (inside <head>) -->
<style>
  p { color: red; }
</style>

<!-- 3. External (recommended) -->
<link rel="stylesheet" href="style.css">
Chapter 02

Syntax & Selectors

Definition: A CSS rule consists of a selector (what to style) and a declaration block (how to style it), containing property–value pairs. 
/* selector { property: value; } */

h1 { color: blue; font-size: 32px; }

/* Element selector */
p { color: gray; }

/* Class selector */
.card { background: #fff; }

/* ID selector */
#header { font-weight: bold; }

/* Universal selector */
* { box-sizing: border-box; }

/* Group selector */
h1, h2, h3 { font-family: sans-serif; }

/* Descendant */
div p { color: green; }

/* Child combinator */
ul > li { list-style: none; }

/* Adjacent sibling */
h2 + p { margin-top: 0; }

/* Attribute selector */
input[type="text"] { border: 1px solid #ccc; }

Chapter 04

Typography

Definition: Typography in CSS controls how text looks — font choice, size, weight, spacing, alignment, and decoration. 
p {
  font-family: 'Inter', Arial, sans-serif; /* font stack */
  font-size: 16px;
  font-weight: 400;        /* 100–900 */
  font-style: italic;
  line-height: 1.6;
  letter-spacing: 0.05em;
  text-align: center;       /* left | right | justify */
  text-decoration: underline;
  text-transform: uppercase; /* lowercase | capitalize */
  color: #333;
  word-spacing: 4px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;  /* truncate with … */
}

/* Google Font import */
@import url('https://fonts.googleapis.com/css2?family=Inter&display=swap');
Chapter 05

The Box Model

Definition: Every HTML element is a rectangular box. The CSS Box Model describes the layers around content: content → padding → border → margin. 
.box {
  width: 200px;
  height: 100px;
  padding: 20px;            /* inside spacing */
  border: 2px solid #7c6dfa;
  margin: 16px;             /* outside spacing */
  box-sizing: border-box;   /* include padding+border in width */
}

/* Shorthand: top right bottom left */
.el { margin: 10px 20px 10px 20px; }
.el { padding: 10px 20px; } /* top-bottom / left-right */

Chapter 06

Display Property

Definition: The display property controls how an element is rendered — whether it takes full width, sits inline, or uses a special layout model. 
.el { display: block; }        /* full-width row */
.el { display: inline; }       /* sits in text flow */
.el { display: inline-block; } /* inline + size control */
.el { display: none; }         /* hidden (removed from layout) */
.el { display: flex; }         /* flexbox container */
.el { display: grid; }         /* grid container */
Chapter 07

Positioning

Definition: The position property controls how an element is placed in the document — relative to its normal position, parent, viewport, or scroll flow. 
.el { position: static; }   /* default: normal flow */

.el { position: relative; top: 10px; left: 20px; }
/* Offsets from normal position, stays in flow */

.el { position: absolute; top: 0; right: 0; }
/* Removed from flow, positioned to nearest relative parent */

.el { position: fixed; bottom: 20px; right: 20px; }
/* Fixed to viewport — stays on scroll */

.el { position: sticky; top: 0; }
/* Sticks when it reaches the top on scroll */

.el { z-index: 10; } /* stacking order */

Chapter 08

Flexbox

Definition: Flexbox (Flexible Box Layout) is a one-dimensional layout model that distributes space and aligns items along a single axis (row or column). 
/* Container properties */
.container {
  display: flex;
  flex-direction: row;          /* row | row-reverse | column */
  justify-content: center;     /* main axis alignment */
  align-items: center;         /* cross axis alignment */
  gap: 16px;
  flex-wrap: wrap;             /* allow wrapping */
}

/* Item properties */
.item {
  flex: 1;                   /* grow and shrink equally */
  flex-grow: 2;              /* take twice the space */
  align-self: flex-end;     /* override container alignment */
  order: -1;                 /* reorder visually */
}

Chapter 09

CSS Grid

Definition: CSS Grid is a two-dimensional layout system that lets you place items into rows and columns simultaneously — ideal for complex page layouts. 
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto;
  gap: 16px;
}

/* Named areas */
.layout {
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
}
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }

/* Span across columns/rows */
.wide { grid-column: span 2; }
.tall { grid-row: 1 / 3; }

Chapter 12

Transforms

Definition: CSS transforms visually move, rotate, scale, or skew elements without affecting document layout. 
.el {
  transform: translateX(50px);   /* move right */
  transform: translateY(-20px);  /* move up */
  transform: rotate(45deg);      /* rotate */
  transform: scale(1.5);         /* scale up */
  transform: scaleX(0.5);        /* shrink horizontally */
  transform: skewX(20deg);       /* skew */

  /* Combine: */
  transform: translate(-50%, -50%) rotate(10deg) scale(1.2);
  transform-origin: top left;   /* pivot point */

  /* 3D transforms */
  transform: perspective(400px) rotateY(30deg);
}
Chapter 13

Transitions

Definition: Transitions smoothly animate a property change from one state to another, triggered by events like hover or focus. 
.btn {
  background: #7c6dfa;
  transition: background 0.3s ease, transform 0.2s ease;
  /* property | duration | timing-function */
}

.btn:hover {
  background: #e05aff;
  transform: scale(1.05);
}

/* Timing functions */
.el { transition-timing-function: ease; }    /* default */
.el { transition-timing-function: linear; }
.el { transition-timing-function: ease-in-out; }
.el { transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1); }
Chapter 14

CSS Animations

Definition: Animations let you define multi-step transitions using @keyframes, running automatically without user interaction. 
/* Define keyframes */
@keyframes slideIn {
  from { opacity: 0; transform: translateY(-20px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%      { transform: scale(1.1); }
}

/* Apply animation */
.card {
  animation: slideIn 0.5s ease forwards;
  /* name | duration | timing | fill-mode */

  animation-delay: 0.2s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-play-state: paused;    /* pause on hover */
}

Chapter 15 · Advanced

CSS Custom Properties (Variables)

Definition: CSS variables (custom properties) store reusable values, defined with --name and accessed with var(--name). They cascade and can be overridden at any scope. 
/* Declare in :root (global scope) */
:root {
  --primary: #7c6dfa;
  --radius: 12px;
  --spacing-lg: 32px;
}

.btn {
  background: var(--primary);
  border-radius: var(--radius);
  padding: 12px var(--spacing-lg);
}

/* Override at component scope */
.dark-card {
  --primary: #e05aff;
}

/* Fallback */
.el { color: var(--brand-color, #000); }

/* JavaScript access */
// getComputedStyle(el).getPropertyValue('--primary')
// el.style.setProperty('--primary', '#fff')

Chapter 16 · Advanced

Pseudo-classes & Pseudo-elements

Pseudo-class: Targets an element in a specific state (e.g., hovered, focused, first child).
Pseudo-element: Targets a part of an element (e.g., the first line, or generated content before/after it). 
/* Pseudo-classes */
a:hover         { color: purple; }
a:visited       { color: gray; }
a:active        { color: red; }
input:focus     { outline: 2px solid blue; }
input:disabled  { opacity: 0.5; }

li:first-child  { font-weight: bold; }
li:last-child   { border-bottom: none; }
li:nth-child(2n){ background: #f0f0f0; } /* even rows */
p:not(.skip)    { color: black; }

/* Pseudo-elements (double colon) */
p::first-line   { font-weight: bold; }
p::first-letter { font-size: 2em; float: left; }
div::before     { content: "→ "; color: purple; }
div::after      { content: " ✓"; }
::selection     { background: #7c6dfa; color: #fff; }
::placeholder   { color: #aaa; }
Chapter 17 · Advanced

Specificity & the Cascade

Definition: The cascade determines which CSS rule wins when multiple rules target the same element. Specificity is the scoring system used to rank selectors.
SelectorSpecificity Score
Inline style1-0-0-0
ID (#id)0-1-0-0
Class / Pseudo-class / Attribute0-0-1-0
Element / Pseudo-element0-0-0-1
!importantOverrides all
p             { color: black; }     /* 0-0-0-1 */
.text         { color: blue; }      /* 0-0-1-0 — wins */
#main .text p { color: red; }       /* 0-1-1-1 — highest */

/* Cascade order (low → high): browser defaults →
   external CSS → internal CSS → inline → !important */
.text { color: green !important; } /* use sparingly */
Chapter 18 · Advanced

Responsive Design

Definition: Responsive design adapts a layout to different screen sizes using fluid units, flexible grids, and media queries. 
/* Mobile-first approach */
.container { padding: 16px; }

/* Tablet */
@media (min-width: 768px) {
  .container { max-width: 720px; margin: 0 auto; }
}

/* Desktop */
@media (min-width: 1024px) {
  .container { max-width: 1100px; }
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  body { background: #0d0f14; color: #e2e5f0; }
}

/* Reduce motion for accessibility */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}

/* Orientation */
@media (orientation: landscape) { ... }

/* Print */
@media print { nav { display: none; } }

Chapter 19 · Advanced

CSS Functions

/* calc() — math with mixed units */
.el { width: calc(100% - 32px); }

/* clamp(min, preferred, max) — fluid sizing */
h1 { font-size: clamp(1.5rem, 4vw, 3rem); }

/* min() / max() */
.col { width: min(100%, 600px); }

/* var() — CSS variable */
.el { color: var(--accent, #7c6dfa); }

/* env() — environment variable */
.safe { padding-bottom: env(safe-area-inset-bottom); }

/* counter() — auto-numbering */
body { counter-reset: section; }
h2::before {
  counter-increment: section;
  content: counter(section) ". ";
}

/* attr() — use HTML attribute in content */
a::after { content: " (" attr(href) ")"; }
Chapter 20

Best Practices & Tips

Always use box-sizing: border-box

*, *::before, *::after { box-sizing: border-box; }

Use CSS custom properties for design tokens

:root {
  --clr-primary: #7c6dfa;
  --space-md: 1rem;
  --radius: 8px;
}

Prefer relative units (rem, em, %) over px

body { font-size: 100%; }        /* = 16px */
h1   { font-size: 2rem; }        /* = 32px */
.card { padding: 1.5em; }

Mobile-first media queries

/* Write base styles for mobile, add breakpoints up */
@media (min-width: 768px) { /* tablet+ */ }
@media (min-width: 1024px) { /* desktop+ */ }

Avoid !important — improve specificity instead

/* Bad */
.btn { color: red !important; }

/* Good — increase specificity */
.theme .btn { color: red; }

Accessibility — focus states

:focus-visible {
  outline: 3px solid #7c6dfa;
  outline-offset: 2px;
}
Summary Checklist: Use box-sizing: border-box globally · Prefer rem/em/%/vw over px · Use CSS variables for tokens · Write mobile-first · Avoid !important · Always define :focus styles · Use clamp() for fluid type · Prefer Flexbox/Grid over floats · Group related properties · Use meaningful class names.
